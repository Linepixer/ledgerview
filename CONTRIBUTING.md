# Contributing to LedgerView

Thank you for your interest in contributing to LedgerView!

LedgerView is an open-source portfolio tracker focused on supporting
investment tracking for users in markets like Argentina, where assets
such as CEDEARs, cryptocurrencies, and local instruments coexist.

This document describes how to contribute to the project.

------------------------------------------------------------------------

## Development Philosophy

LedgerView aims to be:

-   Simple
-   Transparent
-   Reliable for financial tracking
-   Open to community contributions

The project follows clean architecture principles and aims to maintain
high code quality.

------------------------------------------------------------------------

## Getting Started

1.  Fork the repository
2.  Clone your fork locally

``` bash
git clone https://github.com/YOUR_USERNAME/ledgerview.git
```

3.  Create a new branch for your change

``` bash
git checkout -b feature/my-feature
```

------------------------------------------------------------------------

## Commit Style

This project follows **Conventional Commits**.

Format:

    type(scope): message

Examples:

    feat(api): add portfolio endpoint
    fix(transactions): correct asset quantity calculation
    docs: update architecture documentation
    test(api): add endpoint tests
    chore: update project configuration

Common types:

-   `feat` -- new feature
-   `fix` -- bug fix
-   `docs` -- documentation changes
-   `test` -- tests
-   `refactor` -- code improvements
-   `chore` -- project maintenance

------------------------------------------------------------------------

## Pull Requests

When opening a pull request:

-   Provide a clear description of the change
-   Reference related issues if applicable
-   Ensure the code builds and runs correctly
-   Add tests if the change introduces new functionality

------------------------------------------------------------------------

## Code Style

General guidelines:

-   Keep functions small and readable
-   Prefer explicit over implicit logic
-   Document complex logic
-   Follow existing project structure

------------------------------------------------------------------------

## Reporting Issues

If you find a bug or want to request a feature:

1.  Open an issue
2.  Describe the problem clearly
3.  Include steps to reproduce if applicable

------------------------------------------------------------------------

## Code of Conduct

Please be respectful and constructive when interacting with other
contributors.

We want LedgerView to be a welcoming open-source project.

------------------------------------------------------------------------

Thank you for helping improve LedgerView.
